# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **wispy**, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please contact the project maintainer directly via [GitHub](https://github.com/mka-codelake) with:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact assessment
4. Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Resolution target**: Within 30 days (depending on severity)

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Previous minor | Best effort |
| Older versions | No |

## Disclosure Policy

We follow coordinated disclosure. We ask that you give us reasonable time to address the issue before any public disclosure.
